## M2 — 생산 seam 확정 + Mech-Viz cell 정교화 + 시뮬 품질 향상 (4월)

### 무엇을 하나

M1에서 테스트 seam(U1/U2)으로 전체 파이프라인 동작을 검증했으므로, M2에서는:
1. **생산 seam 정의**: CloudCompare point picking → CAD edge 추출로 전환 (Inventor/STEP 기반)
2. **Mech-Viz cell 정교화**: 토치 충돌 모델, 지그/테이블/장애물 추가, TCP 정확도 향상
3. **시뮬 품질 향상**: 접근/이탈 경로 추가, 구간별 충돌/도달불가/flip 검사
4. **추가 캡처 검증**: 새 촬영 데이터에서 생산 seam으로 end-to-end 재검증

> M1에서 이미 달성: Mech-Viz 시뮬 기본 동작 (U1+U2 충돌 0), pose_to_viz.py 변환, GUI→Viz export/시뮬 연동

### 왜 중요한가

테스트 seam으로 "파이프라인이 동작한다"는 증명했지만, 생산용으로 쓰려면 **정확한 용접선 위치**가 필요하다. cell 정교화 없이 실로봇에 넣으면 충돌 위험이 있다.

### 산출물

| 파일 | 내용 |
|------|------|
| `seam_definitions_production.json` | CAD edge 기반 생산 seam 좌표 |
| `sim_cell_config.md` | 시뮬 환경 설정 기록 (로봇 모델, 툴 스펙, 장애물 정보) |
| `SIM_REPORT.md` | 생산 seam 기준 시뮬 검증 결과 |
| 시뮬 영상/스크린샷 | 검증 증거 (포트폴리오용) |

### 완료 기준

- [ ] 생산 seam 최소 2개 CAD edge 기반으로 정의 완료
- [ ] Mech-Viz cell: 토치 충돌 모델 + 주요 장애물 반영
- [ ] 생산 seam 기준 시뮬: 충돌 0건, 도달불가 0건, flip 5° 이내
- [ ] 새 캡처 데이터 1개 이상에서 생산 seam end-to-end 통과

  - 완료조건: 시뮬 환경이 현장과 유사하게 구성

## M3 — 실제 로봇 1회 실행 (5월)

### 무엇을 하나

M2에서 시뮬로 검증된 경로를 **실제 로봇에 넣어서 동작 확인**.

**시나리오 A (협조 가능한 경우)**
- 현장 또는 외부 협조를 통해 실제 로봇에 경로를 넣고 1회 동작 확인
- 두 카메라 경로 중 시뮬에서 더 안정적이었던 것을 우선 사용

**시나리오 B (협조가 어려운 경우)**
- 누구든 와서 30~60분 안에 실행해볼 수 있는 **현장 실행 패키지** 완성
- 실로봇은 다음 기회로 미루고, 시뮬+패키지로 이 단계 완료 처리

### 왜 중요한가

시뮬은 실제 환경을 완벽히 재현하지 못한다. 실로봇 1회만 돌려봐도 시뮬에서 못 잡은 문제(진동, 케이블 간섭, 캘리브레이션 오차 등)가 나온다.

| `FIELD_TEST_PACK.md` | 1페이지짜리 현장 실행 절차 (준비물, 실행 순서, 예상 로그, 실패 대응) |

## M4 — 경로 안정성 확인 + 카메라 A/B 최종 선정 + 파이프라인 확정 (6월)

### 무엇을 하나

1. **반복 안정성 확인**: 같은 조건에서 5회 촬영 → 결과 편차 측정
2. **카메라 A/B 최종 선정**: 3~5월 데이터(정합 품질, 시뮬 통과율, 실로봇 결과)를 종합해 최종 결정
3. **파이프라인 확정**: 선정된 카메라 기준으로 불필요한 분기 제거 + E2E 1회 재현

> 원래 M5(7월)였던 카메라 선정을 6월로 앞당겨 7월은 문서화에만 집중한다.

### 완료 기준

- [ ] 5회 반복 실행 시 RMS ≤ 0.5mm, max ≤ 1.0mm
- [ ] 카메라 선정 문서 1개 (수치 비교 + 근거)
- [ ] 확정 파이프라인 기준 E2E 1회 재현 성공

### 주 단위 WBS (M4)

- `M4-W1` 반복 촬영/생성 5회
  - 완료조건: 반복 RMS ≤ 0.5mm, max 편차 ≤ 1.0mm
  - 재시도: 고정 촬영 환경 상태 재점검 후 1회 재수행
- `M4-W2` 카메라 A/B 최종 선정
  - 완료조건: M1~M3 데이터 종합 점수표 + 선정 문서
  - 재시도: 지표 가중치 1회 조정 후 재산출
- `M4-W3` 파이프라인 확정 + E2E 재현
  - 완료조건: 단일 파이프라인으로 E2E 통과 + 불필요 분기 제거
  - 재시도: 분기 1개씩 제거 후 재실행
