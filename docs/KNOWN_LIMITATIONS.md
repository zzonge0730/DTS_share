- 복수 캡처로 교차 검증 필요


## 2. 정합(ICP) 제약

### 정합 성공 조건

- 촬영 각도: **75도 기준**, HDR 켜짐, 외부 조명 차단
- 검증 범위: 12건 캡처 정상 (fitness 0.554~0.571)
- 실패 사례: 시야각 벗어남 (fitness=0.466), 45도 정면 촬영 (fitness=0.0)

### 용접선 영역 정밀화 부작용

- 용접선 영역 정밀화 적용 시 전체 fitness가 0.91→0.57로 하락 (정상 동작)
- 대신 용접선 영역 정밀도는 향상 (평균 0.9→0.7mm)
- 품질 판정 기준이 정밀화 적용 상태 기준으로 설정되어 있음

### 초기값(seed) 의존성

- 정합 초기값으로 기존 캡처의 변환 행렬을 사용
- 현재 seed: 741, 978 (자동 선택)
- 완전히 새로운 환경에서 seed가 없으면 초기 정합 실패 가능


## 3. 좌표계 / 캘리브레이션

### Hand-Eye 캘리브레이션 없음

- 카메라→로봇 변환이 위치 오프셋만 적용 (회전 변환 미적용)
- 시뮬레이션에서는 문제 없지만, **실제 로봇 실행 시 반드시 필요**
- Mech-Eye SDK에 내장 캘리브레이션 기능 있음

### 로봇 베이스 오프셋

- 현재값: (1.3, -0.5, 0.0)m — 수동 측정값
- 로봇 설치 위치 변경 시 재측정 필요

### 오일러 각도 규칙

- sxyz (고정축 XYZ = 회전축 ZYX) — Mech-Mind + 현대로봇 표준
- 다른 로봇 적용 시 규칙 확인 필요


## 4. 경로 생성 기술 한계

### 표면 스냅 편향

- 최근접 이웃 평균이 점 밀도 높은 평탄면 쪽으로 끌려감
- 모서리 용접에서 1~2mm 편향 가능
- constrained snap으로 일부 완화 (U2에 적용 중)

### 등간격 리샘플링

- 10mm 고정 간격 — 곡선부에서 각진 경로 발생 가능
- 곡률 기반 가변 간격 미구현

### 법선 추정

- 반경 20mm 고정 — 모서리에서 양쪽 면의 점이 섞여 법선 불안정
- 용접 규격(끌림각, 작업각 등) 미반영


## 5. 환경 제약

### 데이터 경로

- 레지스트리 JSON에 원래 개발 환경 절대경로가 기록됨
- `dts_config.json`의 `data_root` 설정 시 자동 변환
- **`dts_config.json` 없이는 원래 개발 PC에서만 동작**

### Mech-Viz 연동

- `scripts/mechviz_runtime.py`에 프로젝트 경로가 일부 하드코딩
- Mech-Viz가 없는 환경에서는 경로 생성까지만 실행 가능
- 시뮬레이션 검증은 Mech-Viz 설치 환경에서만 가능

### WSL 관련

- GUI 폰트: Windows 폰트 디렉토리 탐색 (없으면 기본 폰트 사용, 문제 없음)
- 카메라 촬영: WSL에서 Mech-Eye SDK 사용 시 네트워크 설정 필요

### 검증 범위

- 단일 제품(배터리 케이스) 1개 모델만 검증
- 다른 제품/로트 간 편차는 미검증
- 단위 테스트 131개 전부 통과 (데이터 파일 없이 실행 가능)
